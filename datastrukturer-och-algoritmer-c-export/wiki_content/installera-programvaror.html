Behov

Den programvara ni kommer att behöva på denna kurs är följande:

  1. En editor, i princip vilken som helst, t.ex. atom.
  2. En C-kompilator. Det finns många användbara kompilatorer, men vi kommer att testa er kod med kompilatorn gcc.
  3. En debugger. Det finns många bra debuggers som går att använda.
  4. Ett verktyg för att upptäcka minnesläckor och andra minnesprobelm. Här finns det bara ett bra verktyg som heter valgrind. Programmet valgrind finns tillgängligt under linux och (nyligen) i Windows via WSL2 (Windows Subsystem for Linux 2).

Ni får i princip använda vilken programvara ni vill för att utveckla er kod, men vi kommer att kompilera den med kompilatorn gcc och testa för minnesläckor med verktyget valgrind, i bägge fallen under linux. Vi rekommenderar därför starkt att ni testar koden med gcc+valgrind innan ni lämnar in den.

Alternativ

Vi ser följande alternativ för er:

  1. Sitta vid datorerna i institutionens linux-labb i MA136, MA146, MA156, etc.
    • Denna möjlighet är begränsad just nu då antalet platser i labben är reducerade pga universitetets Covid-19-restriktioner. I skrivande stund finns det tre gånger så många anmälda studenter som platser i labben.
  2. Utveckla på er egen dator hemma och testköra koden på CS datorer innan ni lämnar in den
    • I detta fall kan ni i princip använda vilken kompilator/utvecklingsmiljö ni vill hemma och sen ladda upp filerna till CS filsystem för att testa innan ni lämnar in labben.
    • Inloggning sker då med programmen ssh eller putty.
    • För filöverföring kan ni använda scp eller winscp.
    • Se sektionen Förenklad inloggning för att förenkla processen.
  3. Installera programvarorna på er egen dator
    • Här beror lösningarna på vilket operativsystem ni har på er egen dator
      Linux
      Kör följande kommandon
      1. sudo apt update
      2. sudo apt install gcc make nemiver valgrind
      3. sudo snap install atom --classic
      4. Klart!
      Windows
      Installera programmen under Windows Subsystem for Linux 2 (WSL2)
      1. Installera WSL2 enligt https://docs.microsoft.com/en-us/windows/wsl/install
      2. Kör sedan kommandona under Linux-installationen ovan.
      MacOS
      Installera linux under virtualbox
      1. Installera virtualbox
      2. Installera linux (vi rekommenderar distributionen ubuntu 20.04 lts) under virtualbox.
      3. Kör sedan kommandona under Linux-installationen ovan.

Rekommendationer

Då antalet lab-platser för närvarande är begränsade rekommenderar vi att ni installerar program på er egen dator. Vi rekommenderar i första hand att ni väljer att köra linux, i andra hand att ni kör WSL2 under Windows. Om ni kör MacOS får ni välja mellan lösning 2) och 3).

Förenklad inloggning

Om ni kör ssh för inloggning och scp för filöverföring så går det att förenkla så att ni slipper mata in användarnamn och lösenord hela tiden. Detta är speciellt viktigt om ni väljer lösning 2) ovan.

Gör i så fall följande:

  1. Skapa config-fil på hemmadatorn

    Skapa en fil ~/.ssh/config på er hemmadator med följande innehåll:

    Host peppar salt itchy scratchy
    Hostname %h.cs.umu.se
    User CSusername

    Byt ut CSusername med ert användarnamn på datavetenskap.

  2. Skapa ett nyckelpar på hemmadatorn

    Skapa ett nyckelpar på er hemmadator utan lösenord.

    Kör

    ssh-keygen

    och tryck enter-enter-enter.

    Detta kommer att skapa ett s.k. nyckelpar med en privat och publik nyckel som kan användas till att autenticera sig mot en annan dator.

  3. Kopiera över den publika nyckeln till CS

    Kör

    ssh-copy-id peppar

    och följ instruktionerna.

    Detta tar din nya publika nyckel och lägger den sist i filen .ssh/authorized_keys på CS filsystem.

  4. Testa att logga in

    På din hemmadator, skriv

    ssh peppar

    Nu borde du bli inloggad på peppar utan att behöva skriva varken användarnamn eller lösenord.

  5. Testa att kopiera filer

    På din hemmadator, skriv

    scp filnamn peppar:~

    Detta bör kopiera filen 'filnamn' till din hemmakatalog (~) på CS filsystem.
